BeanDev: UML Plugin von Sourcen für NB 6.9.1 erstellen

(a short English tutorial at the end)

Moin!

Ich sehe ständig auf Twitter, dass alle das UML Plugin vermissen. Seit einigen Teilversionen ist das UML Projekt aus dem Standard geworfen worden. Inzwischen gibt es aber eine Community, die sich um das UML Plugin kümmert. Allerdings gibt es z.Z. keine offiziellen Binaries.

Wer NetBeans mit Java SE installiert hat, kann aber erstaunlich schnell sich selbst das UML-Plugin erstellen. 

Zunächst holt man sich die Quelldateien aus dem Mercurial Repository. Dafür ruft man das Team-Menü -> Mercurial -> Anderes klonen... auf. (Achtung: Mercurial muss installiert sein: HowTo).

Als URL für das Repository, wird diese eingegeben:

http://hg.netbeans.org/community-uml

Benutzer und Kennwort bleiben leer. Auf der nächsten Seite des Assistenten bleiben die Pfade so, wie sie sind. Die Seite darauf bestimmt den Zielordner lokal auf dem Rechner. Das füllt man passend aus. Mit [Beenden] wird nun das Repository heruntergeladen (was ein paar Minuten dauern kann), danach erscheint eine Maske mit der Auswahl der Projekte.

UML [community-uml] ist die Plugin Suite, die das passende Build-Script hat. Dort wird erst mal ein Clean&Build durchgeführt. Da weitere Daten dadurch heruntergeladen werden, kann das je nach Bandbreite erneut ein paar Minuten dauern. Die Warnings im Ausgabe-Fenster kann man ignorieren.

Wenn das alles geklappt hat, öffnet man im Projekte Fenster auf UML [community-uml] das Kontextmenü mit der rechten Maustaste. In diesem Menü: "NBMs erzeugen" aufrufen. Dieses Ant Target hat nach ein bis zwei Minuten die Installationsdateien für das UML-Plugin erzeugt.

Das Ausgabefenster zeigt die erzeugten Dateien mit den Pfaden an, idR. liegen die NBMs unter [...]\{gewählter-Projektordner}\community-uml\build\updates\

Die update.xml Datei ist nur für ein eigenes Update-Center. Nur die NBMs sind spannend.

Jetzt im Menü Extras - Plugins öffnen. Registerzunge |Heruntergeladen| auswählen. Den Button [Plugins hinzufügen...] anklicken und zu den obigen Ordner wechseln. Nun alle NBM Dateien auswählen und hinzufügen.

Die Installationsdateien erscheinen nun im Plugin-Dialog mit einem Haken (fangen alle mit UML... an). Der [Installieren] Button unten links ist aktiv und kann nun gedrückt werden.

Der Installationsassistent will noch bestätigt haben, dass man die Lizenz bestätigt und das fehlende Zertifikat ignorieren möchte. Wenn alles installiert ist, braucht man keinen Neustart.

Mit Datei -> Neues Projekt... erhält man in dem Assistenten die neue Kategorie UML.

Viel Spaß bei ausprobieren.

Wer das UML-Plugin so installiert hat, bekommt natürlich niemals Update-Nachrichten. Man muss die Sourcen dafür aktualisieren und "NBMs erzeugen" erneut ausführen. Wechselt man in den Plugins Dialog, wird i.d.R. automatisch erkannt, dass die Installationsdateien ein Update sind. Wenn nicht. UML deinstallieren und erneut installieren.

Support kann ich nicht geben. Und ggf. kann ein Build mal fehlschlagen. Dann einfach mal einen Tag später mit neueren Sourcen versuchen. Das Projekt ist aktiv in Entwicklung.

Beste Grüße,
  Josch.

 

A short English instruction:

  • You need NetBeans 6.9.1 with JavaSE
  • Please install Mercurial: How-To
  • Open Team -> Mercurial -> Clone other...
  • Repository: http://hg.netbeans.org/community-uml
  • Keep User/Password clean [Next], [Next]
  • Enter a local path [Finish]
  • Wait, ...
  • On UML [community-uml] run Clean&Build
  • Wait, ...
  • On UML [community-uml] run Create NBMs (via context menu)
  • You find the installation files under ...\community-uml\build\updates\
  • Open Plugins -> Downloaded. Click [Add Plugins...]
  • Add all the ...\community-uml\build\updates\*.nbms
  • Click on [Install] (bottom left in your Plugin dialog)
  • Confirm all questions
  • W/o restart you are able to create a "New Project... -> UML"
